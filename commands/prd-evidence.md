Build a complete customer evidence section for a PRD or product proposal using feedback from ~~feedback tool.

Topic: $ARGUMENTS

Apply the `customer-evidence-synthesis` skill throughout. The goal is a structured argument, not a data dump.

Produce the following sections in order:

**1. Problem Statement**
One sentence stating what customers can't do or what's going wrong. Write it from the customer's perspective ("Customers cannot X, which means they Y").

**2. Scale**
- Distinct accounts affected (not ticket count)
- Account tier distribution
- Trend direction (growing, stable, declining) with the time window stated

**3. Sentiment**
How frustrated are customers? Is this mild friction, significant frustration, or a churn-level blocker? Cite sentiment data if available.

**4. Segment Pattern**
Does this affect all customers equally, or is it concentrated in a particular tier, geography, use case, or cohort? If concentrated, note which segment and by how much.

**5. Supporting Quotes**
5 curated quotes selected for diversity (see `customer-evidence-synthesis` for selection rules):
- Enterprise-tier representative
- SMB/self-serve representative
- Recent quote (last 30 days)
- Quote describing a specific workflow impact (not just frustration)
- Quote from a different source channel (if multiple channels available)

Each quote: "[quote text]" — [account tier], [date], [source channel]

**6. Impact Assessment**
HYPOTHESIS: what does this problem cost the business? Consider: churn risk, expansion blocker, competitive disadvantage, support cost. Label all inferences as HYPOTHESIS.

**7. Limitations**
What this evidence section doesn't tell you: coverage gaps, what segments aren't represented, what validation would be needed before treating this as definitive.
